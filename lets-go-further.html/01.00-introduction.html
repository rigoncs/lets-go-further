Chapter 1.

Introduction

In this book we’re going to work through the start-to-finish build of an application called Greenlight — a JSON API for retrieving and managing information about movies. You can think of the core functionality as being a bit like the Open Movie Database API.

Ultimately, our Greenlight API will support the following endpoints and actions:

Method URL Pattern Action
GET /v1/healthcheck Show application health and version information
GET /v1/movies Show the details of all movies
POST /v1/movies Create a new movie
GET /v1/movies/:id Show the details of a specific movie
PATCH /v1/movies/:id Update the details of a specific movie
DELETE /v1/movies/:id Delete a specific movie
POST /v1/users Register a new user
PUT /v1/users/activated Activate a specific user
PUT /v1/users/password Update the password for a specific user
POST /v1/tokens/authentication Generate a new authentication token
POST /v1/tokens/password-reset Generate a new password-reset token
GET /debug/vars Display application metrics

Behind the scenes, we’ll use PostgreSQL as the database to persistently store all the data. And at the end of the book, we’ll deploy the finished API to a Linux server running on DigitalOcean.

Important: The Go 1.25 release ships with experimental support for a new encoding/json/v2 package, which can be enabled by setting the environment variable GOEXPERIMENT=jsonv2 at build time.

This new package brings some nice improvements to working with JSON in Go, but because it is still experimental, the behavior and design of the package is still being evaluated and remains subject to change. For that reason, in this book we will use the stable encoding/json package (which will remain “forever supported” due to the Go compatibility promise).

Once encoding/json/v2 is no longer experimental, and has graduated to general availability in the standard library, I’ll update this book to use it.
